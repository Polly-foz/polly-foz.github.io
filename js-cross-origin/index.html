JavaScript 跨域

Written by Polly-foz with ♥ on January 27, 2020 updateAt Jan 27, 2020 in JavaScript

1. 什么是同源

如果两个页面的协议，端口（如果有指定）和主机都相同，则两个页面具有相同的源。我们也可以把它称为“协议/主机/端口 tuple”，或简单地叫做“tuple”. (“tuple” ，“元”，是指一些事物组合在一起形成一个整体，比如（1，2）叫二元，（1，2，3）叫三元)

http://store.company.com/dir/page.html同源检测的示例:

URL 结果 原因
http://store.company.com/dir2/other.html 成功  只有路径不同
http://store.company.com/dir/inner/another.html 成功  只有路径不同
https://store.company.com/secure.html 失败 不同协议 ( https和http )
http://store.company.com:81/dir/etc.html 失败 不同端口 ( http:// 80是默认的)
http://news.company.com/dir/other.html 失败 不同域名 ( news和store )

2. 什么是跨域

不同源即跨域。当一个资源从与该资源本身所在的服务器不同的域、协议或端口请求一个资源时（两者不符合同源策略），资源会发起一个跨域 HTTP 请求（CORS request）。 1中表格的后三个例子就是跨域。

3. JSONP跨域

Jsonp(JSON with Padding) 是跨域的一种解决方案。JSONP 的原理很简单，就是利用 <script> 标签没有跨域限制的漏洞。通过 <script> 标签指向一个需要访问的地址并提供一个回调函数来接收数据。

假如http://data-requester.com:9999想要http://data-owner.com:8888数据库中的friends.json文件，

http://data-owner.com:8888服务端node.js代码：

if(path === '/friends.js'){
    //检查资源请求者的资格
    if(request.headers['referer'].indexOf('http://data-requester.com:9999') === 0 ){//拥有资格，返回JavaScipt脚本，调用资源请求者提供的回调函数，并将数据作为回调函数参数传入
        response.setHeader("Content-Type","text/javascript;charset=utf-8")
        const data = fs.readFileSync('./db/friends.json').toString()
        response.write(`window[${query.callback}](${data})`)
        response.end()
    }else{//不拥有资格，返回404
        response.statusCode = 404
        response.end()
    }
  }

资源请求者在script标签的src属性指定请求的地址，通过callback参数传递回调函数的名字。 http://data-requester.com:9999客户端JavaScript代码：

<script>
const jsonp = (url)=>{
    return new Promise((resolve,reject)=>{
        let script = document.createElement("script")
        const random = 'callbackFunctionName' + Math.random()//生成回调函数名
        window[random] = data=>{//将回调函数添加到全局的window对象上
            resolve(data)
        }
        script.src = `${url}?callback=${random}`
        document.body.appendChild(script)
    })
}
//在控制台打印出获取的数据
jsonp("http://data-owner.com:8888/friends.js").then(data=>{console.log(data)})
<script>

缺点

JSONP只支持GET的方式，并不支持POST方式。

4. CORS跨域

同样的例子，http://data-requester.com:9999想要http://data-owner.com:8888数据库中的friends.json文件，

资源拥有者添加Access-Control-Allow-Origin字段，设置允许的源。 http://data-owner.com:8888服务端node.js代码：

if(path === '/friends.json'){
    response.statusCode = 200
    response.setHeader("Content-Type","text/json;charset=utf-8")
    response.setHeader("Access-Control-Allow-Origin","http://data-requester.com:9999")
    response.write(fs.readFileSync("./db/friends.json"))
    response.end()
}

资源请求者发送AJAX请求

function cors(){
    let request = new XMLHttpRequest()
    // request.setRequestHeader("Content-Type","application/json")
    request.open('GET','http://data-owner.com:8888/friends.json')
    request.onreadystatechange = ()=>{
        if(request.readyState === XMLHttpRequest.DONE && status >= 200 && states < 400){
            console.log(request.response)
        }
    }
    request.send()
}
cors()

优点

CORS支持所有类型的HTTP请求。

缺点

从CORS兼容性看出，CORS不兼容IE7及以下。
