1. 浏览器渲染原理

渲染，简单说，就是浏览器将html，css，js源代码转化为用户可以看见的网页。

1.1. 浏览器渲染过程

  1. 根据HTML构建HTML树（DOM）
  2. 根据CSS构建CSS树（CSSOM）
  3. CSSOM 树和 DOM 树合并成渲染树（Render Tree）
    • 渲染树只包含渲染网页所需的节点(可见节点)。
      • display:none元素从渲染树中完全移除
      • visibility:hidden仍占据着布局空间，被渲染成一个空框
    • 最终输出的渲染同时包含了屏幕上的所有可见内容及其样式信息。
  4. 布局计算每个对象的精确位置和大小。
  5. 绘制，将渲染树转换成屏幕上的实际像素。
  6. 合成，根据层叠关系展示画面

1.2. 更新样式

1.2.1. 如何更新样式

  • 一般用JS来更新样式
    • e.g. div.style.background = ‘red’;
    • e.g. div.style.display = ‘none’
    • e.g. div.classList.add(‘red’)
    • e.g. div.remove()
  • 其它方法有CSS Animations、Transitions 和 Web Animation API等。
  • 浏览器会重新渲染以更新样式
  • 对于不同的样式，浏览器重新渲染的方式也有所不同，有以下三种更新方式。

1.2.2. 三种更新方式

  1. JS / CSS > 样式 > 布局 > 绘制 > 合成
    • 修改了元素的“layout”属性，即改变了元素的几何属性（例如宽度、高度、左侧或顶部位置等）。
    • e.g. div.remove()
  2. JS / CSS > 样式 > 绘制 > 合成
    • 修改“paint only”属性（例如背景图片、文字颜色或阴影等），即不会影响页面布局的属性。
    • e.g. background
  3. JS / CSS > 样式 > 合成
    • 更改一个既不要布局也不要绘制的属性。
    • e.g. transform
  • JS/CSS
    • 使用JavaScript或CSS等实现视觉变化的效果
  • 样式计算
    • 根据匹配选择器计算元素的最终样式
  • 绘制
    • 将元素转换为像素

  • 合成

    • 根据层叠关系，将元素按正确顺序绘制到屏幕上

  • 如果想知道更改任何指定 CSS 属性将触发上述三种方式中的哪一个，可以访问CSS触发器。

2. CSS 动画的两种做法（transition 和 animation）

CSS3使得前端开发人员能够不借助JavaScript或Flash就能在HTML和CSS中设计一些动画效果。 * CSS3 transitions让开发人员能够在元素状态改变（如悬停，聚焦等）时，转换一个元素的外观和行为，并且为转换添加过渡。在此之前，状态改变都是即时生成的。 * CSS3 animations让元素能在多个关键帧上改变外观和行为。

2.1. Transition（过渡）

2.1.1. 改变状态的方式

使用伪类 * :hover * :focus * :active * :target

2.1.2. 可以设置的过渡相关属性

  • transition-property 声明要过渡的属性
    • 不同状态的属性都会被转换，但是只有在transition-property声明的属性的转换会被过渡。
    • 不是所有的属性都能被过渡，只有具有可以计算出中间状态的属性才能被过渡。
    • e.g. display，opacity属性很可能不能被过渡。
  • transition-duration 指定过渡的持续时间
    • 可以对每个过渡属性各声明一个过渡的持续时间
  • transition-delay 指定在过渡效果开始作用之前需要等待的时
    • 可以对每个过渡属性各声明一个过渡的延迟
  • transition-timing-function 指定过渡的移动速度
    • 取值：
    • linear 匀速
    • ease-in 加速
    • ease-out 减速
    • ease-in-out 先加速后减速
    • cubic-bezier函数 自定义速度模式
    • 可以对每个过渡属性各声明一个过渡的移动速度
  • transition 上面4个属性的简写属性形式
  • 使用上述属性，需要加浏览器前缀

2.2. Animation

与transition相比，animation有以下优点： 1. 可以不需要事件触发 2. 可以指定动画重复次数 3. 能够方便地指定多个状态，实现更复杂的动画效果

2.2.1. 关键帧

动画关键帧通过@keyframes规则规定，包括动画名称，动画断点和需要做动画的属性。关键帧的一个例子如下：

@keyframes identifier {
  0% { top: 0; left: 0px}
  50% { top: 30px; left: 20px; }
  50% { top: 10px; }
  100% { top: 0; left: 30px;}
}
  • keyframes需要添加浏览器前缀

2.2.2. animation相关属性

  • animation-name 关键帧规则的名称
    • 给元素指派关键帧规则
    • 需要和anmiation-duration共同使用
  • animation-duration 指定一个动画周期的时长
    • 默认值为0s，表示无动画。
  • animation-timing-function 定义CSS动画在每一动画周期中执行的节奏
  • animation-delay 动画播放前的延迟
  • animation-iteration-count 动画播放次数
  • animation-direction 动画播放方向
  • animation-fill-mode 设置CSS动画在执行之前和之后如何将样式应用于其目标
    • 取值：
    • none 动画开始前或结束后，不指派任何样式
    • forwards 动画结束后，目标样式为最后一个关键帧时的样式
    • backwards 元素被指派动画后，将目标样式指定为第一个关键帧时的样式
    • both 应用forwards和backwards
  • animation-play-state 指定动画播放状态
  • animation 上面属性的简写属性形式
