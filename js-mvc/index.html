MVC是什么

  • M - Model（数据模型） 程序要操作的数据和对数据的基本操作
  • V - View（视图） 负责界面的布局和显示
  • C - Controller（控制器） 负责模型和界面之间的交互，用于控制应用程序的流程，处理用户的行为和数据model上的改变

V <–> C <–> M

Model = {
    data: 数据
    get(){return data}
    set(data){
        编辑数据
        触发数据变化事件'data:set'
    }
}

View = {
   el: 界面的挂载点
   html: 界面的内容
   render(){将this.html挂载到el上}
}

Controller = {
    bindEvents(){
        在dom元素上绑定事件监听器
        监听数据变化事件'data:set'(){
            View.render()
        }
    }

}

EventBus

上面模型中，数据变化事件的触发与监听可以通过EventBus实现。

用法

//创建eventBus
//用jquery实现
const eventBus = $(window)

//监听数据变化事件'data:set'
eventBus.on("data:set",()=>{View.render()})

//取消监听数据变化事件'data:set'
eventBus.off("data:set")

//触发数据变化事件'data:set'
eventBus.trigger("data:set")

表驱动编程

表驱动方法是一种使你可以在表中查找信息，而不必用很多的逻辑语句（if或Case）来把它们找出来的方法。

Controller = {
    bindEvents(){
        dom1.on("event1",f1)
        dom2.on("event2",f2)
        dom3.on("event3",f3)
        dom4.on("event4",f4)
        ......
        domn.on("eventn",fn)
    }
    f1(){}
    f2(){}
    f3(){}
    f4(){}
    ...
    fn(){}
}

上面的代码看起来有些许冗余，借用表驱动编程的部分思想，可以使上面的代码看起来没那么冗余。

Controller = {
    events:{
        "event1 dom1":"f1"
        "event2 dom2":"f2"
        "event3 dom3":"f3"
        "event4 dom4":"f4"
        ......
        "eventn domn":"fn"
    }
    bindEvents(){
        for(let key in Controller.events){
            event = key空格前的部分
            dom = key空格后的部分
            View.el.on(event,dom,Controller[Controller.events[key]])
        }
    }
    f1(){}
    f2(){}
    f3(){}
    f4(){}
    ...
    fn(){}
}

模块化

模块化是指将实现特定功能的代码分离成一个独立的js文件。1个js文件就是一个模块。

优点

  1. 便于多人协作开发，各模块可以独立开发
  2. 便于调试
  3. 利于代码复用
  4. 利于功能扩充

缺点

  1. 模块间通信消耗额外性能
