Webpack学习笔记2

Written by Polly-foz with ♥ on February 7, 2020 updateAt Feb 7, 2020 in

1. 用webpack转译JS

mkdir webpack-demo-1
cd webpack-demo-1
npm init -y
yarn add webpack webpack-cli --dev

构建目录结构

 webpack-demo
  |- package.json
  |- yarn.lock
  |- node_modules
+ |- /src
+   |- index.js
+   |- x.js

index.js

import x from './x.js'
console.log('xxx')

x.js

export default 'xxx'

如何调用本地webpack?

./node_modules/.bin/webpack

or

npx webpack

NOTE:

npx免去了我们自己寻找的过程，但是不稳定。

运行上述代码后，目录结构变为：

 webpack-demo
  |- package.json
  |- yarn.lock
  |- node_modules
  |- /src
    |- index.js
    |- x.js
+ |- /dist
+   |- main.js

可以看见main.js尾部代码：

console.log("xxx")

webpack将js转换成更低版本的语法

2. 初始化webpack.config.js

上面代码运行webpack后给出警告，警告内容：

WARNING in configuration
The 'mode' option has not been set, webpack will fallback to 'production' for this value. Set 'mode' option to 'development' or 'production' to enable defaults for each environment.
You can also set it to 'none' to disable any default behavior. Learn more: https://webpack.js.org/configuration/mode/

新建配置文件：

 webpack-demo
  |- package.json
  |- yarn.lock
  |- node_modules
  |- /src
    |- index.js
    |- x.js
  |- /dist
    |- main.js
+ |- webpack.config.js

webpack.config.js (CRM大法，官网找config)

module.exports = {
    mode: 'development'
};

运行webpack后再看main.js，变化好大！

mode取值：production(default) | development

production development
缩小代码体积，优化用户体验 开发者用，给出可看懂的代码各式
