1. CSS 盒模型(Box Model)

CSS 盒模型是对一个文档进行布局时，浏览器渲染元素的标准之一。它将所有的元素表示为一个个矩形的盒子。一个盒子由边距(margin)，边框(border)，填充(padding)和实际内容(content)组成，如下图所示：

  • 内容区域 content area
    • 容纳着元素的“真实”内容，如文本、图像或一个视频播放器。
    • 它通常含有一个背景颜色（默认为透明）或背景图像。
  • 填充区域 padding area
    • 内容区域的扩展，负责延伸内容区域的背景，填充内容与边框之间的区域。
  • 边框区域 border area
    • 扩展自内边距区域，是容纳边框的区域。
  • 边距区域 margin area
    • 用空白区域扩展边框区域，以分开相邻的元素。

2. 两种 CSS 盒模型

根据 width 和 height 计算方式的不同，有下面两种常见的 CSS 盒模型。

2.1. content-box 内容盒（W3C 标准盒模型）

  • content-box width = content width
  • content-box height = content height

2.2. border-box 边框盒（IE 盒模型）

  • border-box width = content width + padding + border width
  • border-box height = content height + padding + border width

2.3. CSS 盒模型的设置

  • 默认的 CSS 盒模型是 content-box。
  • 通过设置 CSS 中的 box-sizing 属性，可以指定 CSS 盒模型。
    • box-sizing 取值：
    • content-box
    • border-box
  • 一般情况下，尤其是实现响应式布局，建议使用 border-box 模型。

3. 行内元素占用空间

除可替换元素外，对于行内元素来说，尽管内容周围存在内边距与边框，但其占用空间（每一行文字的高度）则由 line-height 属性决定，即使边框和内边距仍会显示在内容周围。

4. margin 合并

块级元素的上边距和下边距有时会合并（或折叠）为一个外边距，其大小取其中的最大者，这种行为称为边距折叠（margin collapsing），有时也翻译为边距合并。注意浮动元素和绝对定位元素的边距不会折叠。

4.1. 会发生 margin 合并的情况

  • 相邻元素之间（兄弟合并）
  • 父元素与其第一个或最后一个子元素之间（父子合并）
  • 空的块级元素
    • 前提：
    • 在其 margin-top 与 margin-bottom 之间没有 border、padding、content、height、min-height 将两者分开。
    • 结果：
    • 该元素的上下外边距会折叠。

4.2. 如何阻止合并

  • 父子合并
    • 用 padding/border 挡住
    • or 用 overflow:hidden 挡住
    • or 用 display:flex

  • 兄弟合并

    • 是符合预期的
    • 可以用 inline-block 消除
